# 학기 프로젝트 제안서

## 프로젝트명
On-Device AI 민원 분석 및 처리 시스템

## 문제 정의
공공기관 민원 데이터는 개인정보와 민감 정보가 포함될 수 있어 외부 클라우드로 전송하는 방식은 보안 부담이 크다. 또한 기존의 단순 키워드 분류는 민원 문맥을 충분히 반영하지 못해 유사 사례 검색과 답변 초안 작성의 효율이 낮다. 담당자는 방대한 민원 사례를 직접 확인해야 하므로 처리 시간이 길고, 표준화된 응대 품질도 유지하기 어렵다.

## 핵심 기능 3개
1. 로컬 민원 데이터 수집 및 정제
   - 각 지자체 전자민원창구의 공개 민원과 답변을 수집하고, 개인정보를 제거한 뒤 학습 가능한 형태로 정제한다.
2. 의미 기반 유사 사례 검색
   - 민원 내용을 임베딩하고 FAISS 기반 검색을 통해 과거의 유사 민원과 처리 결과를 즉시 찾는다.
3. 표준 답변 초안 생성
   - 온디바이스 sLLM이 민원 요지를 분석해 표준 서식에 맞는 답변 초안을 생성하고, 담당자가 빠르게 검토·수정할 수 있도록 지원한다.

## 기술 스택
- AI 모델: HuggingFace, Ollama, Gemma 2, Llama 3
- 검색/추론: FAISS, ONNX, TensorRT, Quantization
- 학습: Open Source LLM Fine-tuning, SFT
- Backend: Python, FastAPI
- Frontend: React, TypeScript
- Deployment: 로컬 서버 또는 온프레미스 환경
- Collaboration: Git, GitHub
