# PR Review Guide

이 문서는 리뷰어가 PR을 빠르게 판단하고, 작성자가 바로 수정할 수 있도록 구조화된 피드백 기준을 정의합니다.

## 리뷰 원칙
- 먼저 동작, 안정성, 유지보수성에 영향을 주는 항목부터 본다.
- 단순 취향과 실제 결함을 분리한다.
- 각 코멘트는 수정 가능하고 검증 가능한 주장만 담는다.

## 태그 기준
- `[MUST]`: 병합 전에 반드시 수정해야 하는 문제
- `[SHOULD]`: 병합은 가능하지만 개선하면 좋은 항목

## 코멘트 예시
- `[MUST] 브랜치 보호 규칙과 충돌할 수 있으므로 direct push를 막는 설정을 적용해 주세요.`
- `[MUST] PR 템플릿에 검증 항목이 비어 있어, 재현 가능한 확인 절차를 추가해 주세요.`
- `[SHOULD] 커밋 예시를 한 줄 더 추가하면 팀원이 Conventional Commits를 더 쉽게 적용할 수 있습니다.`
- `[SHOULD] README에 리뷰 순서를 넣으면 신규 참여자가 따라가기 편합니다.`

## 최소 리뷰 기준
- 구조적 문제를 1건 이상 확인한다.
- 사용자 영향 또는 협업 영향이 있는 항목을 1건 이상 확인한다.
- 개선 제안 1건 이상을 남긴다.
