# 공공기관 민원 응답 지원 sLLM 시스템 구축 제안서

## 1. 프로젝트 개요
- **프로젝트명**: 폐쇄망 환경 기반 민원 응답 지원 AI 시스템
- **목표**: 공공기관 민원 데이터를 활용해, 담당 공무원이 빠르고 일관된 답변 초안을 생성할 수 있는 온프레미스 AI 서비스를 구축한다.
- **핵심 방향**: 외부 클라우드 의존 없이 로컬/폐쇄망에서 동작하는 **의미 기반 검색 + 경량 LLM(sLLM) 답변 생성** 체계를 구현한다.

---

## 2. 개발 배경 및 필요성
1. **보안·개인정보 보호 요구**
   - 공공기관 민원 데이터에는 민감한 개인정보가 포함될 수 있어 외부 전송 시 보안 위반 리스크가 높다.
   - 따라서 폐쇄적 로컬 환경(온프레미스/폐쇄망)에서의 데이터 처리 및 모델 운용이 필수적이다.

2. **기존 키워드 기반 방식의 한계**
   - 단순 키워드 매칭은 민원 맥락(의도, 상황, 정책 근거)을 충분히 반영하기 어렵다.
   - 동일 단어라도 문맥에 따라 다른 처리 기준이 필요한 민원 업무 특성상 의미 기반 접근이 필요하다.

3. **업무 효율성 개선 필요**
   - 담당자가 유사 민원 사례를 수작업으로 탐색·비교하는 데 많은 시간이 소요된다.
   - 의미 기반 유사 사례 검색을 통해 과거 답변을 즉시 참조하고, 답변 초안 자동 생성으로 업무 시간을 단축할 수 있다.

4. **경제적 AI 도입 가능성**
   - 대규모 범용 모델 대비, 도메인 특화 sLLM은 운영비용이 낮고 성능 대비 효율이 높다.
   - 기관 단위 예산과 인프라 제약을 고려할 때 현실적인 도입 전략이다.

---

## 3. 필요 기술 스택
- **모델·학습**: HuggingFace, 오픈소스 LLM 파인튜닝(SFT)
- **추론·서빙**: Ollama, ONNX Runtime, TensorRT
- **검색·지식기반**: FAISS(벡터 인덱싱/유사도 검색)
- **최적화**: Quantization(INT8/INT4 등)
- **UI**: Streamlit
- **배포**: Docker(폐쇄망 이식성 고려)

---

## 4. 개발 요구사항 및 구현 방안

### 4.1 데이터 확보 및 학습
- 각 지자체별 전자민원창구의 **공개 민원/답변 데이터 수집 크롤러** 개발
- 수집 데이터 정제(중복 제거, 포맷 통일, 비식별화)
- 오픈소스 sLLM(Gemma 2, Llama 3 등) 대상 **SFT(Supervised Fine-tuning)** 수행
- 학습 데이터셋/모델 버전 관리 및 품질 검증 체계 마련

### 4.2 표준 답변 생성 엔진
- 기관 민원 응답 가이드에 맞춘 **시스템 프롬프트 템플릿** 설계
  - 문체(정중/명확), 구성(요지-근거-안내), 금지표현 등 반영
- 민원 요지 입력 시:
  1) FAISS 기반 유사 사례 검색
  2) 검색 결과 + 표준 프롬프트를 결합한 답변 초안 생성
- 담당자가 즉시 수정·보완 가능한 실무형 출력 제공

### 4.3 고성능 온프레미스 서빙 최적화
- 폐쇄망 설치 가능한 실행 패키지 구성
- 모델 변환(ONNX) 및 TensorRT 기반 추론 가속 적용
- 양자화(Quantization)를 통한 메모리 절감 및 지연시간 최소화
- 운영 환경(CPU/GPU)별 프로파일링 및 성능 기준 수립

### 4.4 사용자 친화적 UI
- Streamlit 기반 웹 인터페이스 제공
  - 민원 입력창
  - 답변 생성 결과창
  - 수정/복사 기능
  - (선택) 유사 사례 참조 패널
- 비전문가도 즉시 사용 가능한 단순·직관적 화면 설계

---

## 5. 개인정보 보호 및 컴플라이언스
- 공개 게시판 데이터만 활용하되, 학습 전 **비식별화(이름/연락처/주소/주민등록식별정보 등 제거)** 강화
- 원문 접근권한/학습 데이터 접근권한 분리
- 로그 내 개인정보 마스킹 정책 적용
- 관련 가이드라인(개인정보 보호, 공공데이터 활용 지침) 준수

---

## 6. 기대 효과
1. **민원 처리 시간 단축**: 유사 사례 검색과 초안 자동 생성으로 반복 업무 감소
2. **답변 품질 표준화**: 기관 톤앤매너와 정책 근거 반영으로 일관성 확보
3. **보안 리스크 최소화**: 외부 클라우드 전송 없는 폐쇄망 운영
4. **비용 효율성 확보**: 도메인 특화 sLLM 기반으로 구축·운영비 절감
5. **확장성**: 타 지자체/타 공공기관으로의 수평 확산 및 이식 용이

---

## 7. 향후 확장 계획
- Docker 기반 컨테이너화로 폐쇄망 환경 간 빠른 배포 지원
- 기관별 템플릿/정책 룰셋 분리 관리로 맞춤형 운영
- 피드백 루프(수정 이력 기반 재학습)로 지속 성능 개선

---

## 8. 결론
본 과제는 공공기관이 요구하는 보안성과 실무 효율을 동시에 충족하는 현실적 AI 도입 모델이다. 
의미 기반 검색, 도메인 특화 sLLM, 온프레미스 최적화 기술을 결합해 **신속·정확·안전한 민원 응답 지원 체계**를 구현할 수 있다.
